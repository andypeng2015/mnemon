Mnemon Harness / Skill Loop

Skill Loop MVP

第一版 skill loop 采用和 memory loop 相同的表达方式：GUIDE 放规则，hook 只给时机，protocol skill 做在线轻量写入，curator subagent 做后台 review 和提案。HostAgent 负责原生 skill 使用，`.mnemon` 提供 canonical skill library；没有单独的 refresh 阶段，可见面变化在下一次 Prime 生效。

阅读顺序

  1. 先看三大核心：HostAgent、Host Skill Surface、Skill Library。
  2. 再看支撑资产：GUIDE、Protocol skills、Usage sidecar、Curator。
  3. 最后切换阶段，看 Prime、Nudge、Observe、Curator、Manage 如何闭合成 loop。

System Components

Skill loop 的核心不是冷热存储，而是可见性治理：哪些 skill 当前可被发现、哪些进入维护、哪些仅保留历史。

Three Core Parts
HostAgent

宿主 Agent 的执行引擎。它拥有 ReAct loop、hook bus、prompt assembly、tool routing 和原生 skill/subagent 调用。

Host Skill Surface

HostAgent 原生 skill 发现面，例如 `.claude/skills`。Prime 将 `.mnemon/skills/active` 单向同步或挂载到这里。

.mnemon Skill Library

canonical filesystem：`skills/active`、`skills/stale`、`skills/archived`。所有真实状态都以它为准。

Maintained Assets
GUIDE

定义什么算 skill usage evidence、reusable workflow signal、何时触发 review、何时只能 proposal。

Protocol skills

`skill_observe.md`、`skill_manage.md`、`skill_curate.md` 定义记录 evidence、迁移状态和启动 review 的协议。

Usage sidecar

`.mnemon/skills/.usage.json` 记录 view/use/patch 计数、时间戳、state、pinned 等治理信号。

Curator subagent

低频维护流程。可手动触发，也可由 HostAgent scheduler 或 compact/dreaming 边界触发。

Active 才影响行为

Prime 只同步 active skills。stale 和 archived 默认不进入 host-native skill surface，只在 review、restore 或人工查询时出现。

Nudge 只提醒遵守 GUIDE

Nudge 是 stop hook 注入的一句话提醒。判断规则在 GUIDE 中，在线写入由 `skill_observe.md` 执行。

Curator 是维护路径

它不是每次任务都执行的在线步骤。没有 scheduler 时可以手动运行；不能安全写入时只产出 proposal/report。

Manage 不直接刷新 Host

skill_manage 只更新 `.mnemon` canonical state。新的 active 集合要等下一次 Prime 再同步到 Host Skill Surface。
