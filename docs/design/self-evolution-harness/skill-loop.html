Mnemon Harness / Skill Loop

Skill Loop MVP

第一版 skill loop 采用和 memory loop 相同的表达方式：先明确实体，再按阶段展示数据流。HostAgent 负责调用与判断，`.mnemon` 提供 canonical skill library，protocol skills 定义读写方法，curator subagent 负责低频治理。

阅读顺序

  1. 先看三大核心：HostAgent、Active Skill Index、Skill Library。
  2. 再看支撑资产：Protocol skills、Usage sidecar、Curator、Safety reports。
  3. 最后切换阶段，看在线使用和维护治理如何通过 active index 闭合。

System Components

Skill loop 的核心不是冷热存储，而是可见性治理：哪些 skill 当前可被发现、哪些进入维护、哪些仅保留历史。

Three Core Parts
HostAgent

宿主 Agent 的执行引擎。它拥有 ReAct loop、hook bus、prompt assembly、tool routing 和原生 skill/subagent 调用。

Active Skill Index

当前可见行为面。Prime/Remind 只暴露 active skills 的轻量索引，不全量加载 skill library。

.mnemon Skill Library

canonical filesystem：`skills/active`、`skills/stale`、`skills/archived`。所有真实状态都以它为准。

Maintained Assets
Protocol skills

`skill_list.md`、`skill_view.md`、`skill_manage.md`、`skill_curate.md` 定义跨 HostAgent 的读写协议。

Usage sidecar

`.mnemon/skills/.usage.json` 记录 view/use/patch 计数、时间戳、state、pinned 等治理信号。

Curator subagent

低频维护流程。可手动触发，也可由 HostAgent scheduler 或 compact/dreaming 边界触发。

Safety reports

protected、pinned、dry-run、backup、restore、rollback 和 `reports/skills` 让治理可审计、可恢复。

Active 才影响行为

stale 和 archived 默认不进入 prompt，也不主动参与 skill discovery。它们只在 review、restore 或人工查询时出现。

Protocol 不写死 Host

所有 protocol skills 读写 `.mnemon` canonical layout；Claude Code、Codex 等 native skill 目录只是 projection。

Curator 是维护路径

它不是每次任务都执行的在线步骤。没有 scheduler 时可以手动运行；不能安全写入时只产出 proposal/report。
