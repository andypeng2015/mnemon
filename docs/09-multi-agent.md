# 09. 多 Agent 记忆维护

---

## 9.1 角色分工

Mnemon 作为独立 daemon 支持多个专门 agent 并行维护记忆：

| Agent | 职责 | 类比 |
|-------|------|------|
| 主 Agent | 日常交互，正常读写记忆 | 玩家在线行为 |
| 整理 Agent | 定期扫描图谱，去重/合并/归纳 | 数据库压缩/GC |
| 质疑 Agent | 检查矛盾和过时信息，标记需更新的记忆 | 数据完整性校验 |
| 关联 Agent | 寻找隐含关系，建立跨领域关联 | 索引构建器 |

---

## 9.2 核心价值

打破了"记忆维护者和使用者必须是同一智能体"的限制。人类大脑中两者确实相同，但这是认知资源竞争的瓶颈。Mnemon 让记忆维护变成**并行的、持续的、多视角的**。

---

## 9.3 风险与对策

多 agent 写同一图谱需要处理冲突和级联错误（分布式一致性问题）：

- 操作原子性
- 版本控制
- 软删除 + 审计日志
- 关键记忆不可变标志
- 置信度标记
